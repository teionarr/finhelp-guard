## What the live run taught us (two real findings)

1. **A grounding gap the offline tests couldn't surface.** On T-1 the live model quoted the
   account balance (`$240.00`) it got from the `lookup_account` tool. The groundedness rail
   originally grounded only against *KB search results*, so it flagged that legitimately-sourced
   figure as unsupported and routed to review. Fix: **tool outputs are grounding evidence too**
   (`agent._tool_facts`) — CRM facts now count. T-1 passes for the right reason.

2. **The judge closes the recall gap but over-refuses — the precision/recall tradeoff, live.**
   The deterministic rails miss every held-out paraphrase (recall 0.000) but never false-refuse
   (0.000). The LLM judge recovers advice- and groundedness-recall to 1.000 — *and* pushes
   false-refusal to 1.000 on the (tiny, n=2) benign held-out set. Neither alone is shippable:
   this is exactly why the harness certifies interval bounds and why a production judge must be
   **threshold-tuned and calibrated against human labels**. The number now has evidence behind it,
   not just a README claim.
